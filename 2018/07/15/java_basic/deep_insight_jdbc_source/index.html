总结

让我们再次从时序图来看看 Java JDBC 源码的整个流程。

从时序图中我们可以了解到，Java JDBC 源码是通过 DriverManager 这个类进行数据接口的开放。而在 Java SPI 机制则是通过 ServiceLoader 类进行特定配置文件的扫描和读取，从而实现框架扩展的功能。
